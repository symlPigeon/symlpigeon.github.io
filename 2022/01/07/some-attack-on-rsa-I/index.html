# RSA

RSA 算法的主要经过这样的一个过程：

选取两个素数ppp 和qqq，计算模数n=p∗qn=p*qn=p∗q，并计算欧拉函数ϕ(n)=(p−1)(q−1)\phi(n)=(p-1)(q-1)ϕ(n)=(p−1)(q−1)。选取一个公钥指数eee 满足gcd(ϕ(n),e)=1gcd(\phi(n), e)=1gcd(ϕ(n),e)=1，计算私钥（解密指数）ddd 满足ed≡1(mod ϕ(n))ed\equiv 1(mod\ \phi(n))ed≡1(mod ϕ(n))。销毁p,q,ϕ(n)p,q,\phi(n)p,q,ϕ(n).

RSA 的公钥为(n,e)(n, e)(n,e)，私钥为(n,d)(n, d)(n,d)。

在加密时，对于明文mmm，计算c≡me(mod n)c\equiv m^e(mod\ n)c≡me(mod n)。在解密时，计算m≡cd(mod n)m\equiv c^d(mod\ n)m≡cd(mod n)。

可以看出，cd(mod n)=med(mod n)=med(mod ϕ(n))(mod n)=m(mod n)c^d(mod\ n)=m^{ed}(mod\ n)=m^{ed(mod\ \phi(n))}(mod\ n)=m(mod\ n)cd(mod n)=med(mod n)=med(mod ϕ(n))(mod n)=m(mod n)，能够满足加解密一致性。

RSA 的安全性主要依赖于大整数分解的问题，也就是，在获取到公钥中的模数nnn 的情况下，是很难分解nnn 的模数p,qp,qp,q，否则很容易计算出私钥，进而破解 RSA 体系。

# 常规的分解思路？

对于比较大的模数来说，直接进行暴力的分解算是非常低效的做法，基本上是不可能的。Fermat 分解是一种算是比较快的做法，当nnn 的因数比较接近nnn 的平方根时，可以较快实现分解。

我们注意到有n=pq=(p+q2)2−(p−q2)2=a2−b2=(a+b)(a−b)n=pq=(\frac{p+q}{2})^2-(\frac{p-q}{2})^2=a^2-b^2=(a+b)(a-b)n=pq=(2p+q​)2−(2p−q​)2=a2−b2=(a+b)(a−b)，其中a=p+q2,b=p−q2a=\frac{p+q}2,b=\frac{p-q}2a=2p+q​,b=2p−q​。

# MFFV2, MFFV3, MFFV4...

MFFV2 and MNQSV2: Improved Factorization Algorithms 给了一个优化的 Fermat 分解算法的思路，主要目的在于减少平方根计算的次数来优化运行时间。这个算法名字记为 MFFV2 (Modified Fermat Factorization Version 2)。

平方数的末位必定是 1, 4, 9, 6, 5, 0 ，如果一个数的个位不是这些数字则其必定不是完全平方数。在 FFA 中，如果a2−na^2-na2−n 的个位不是这些数那么就不需要再开平方根了，这能够节约一定的计算量。

MFFV2 的循环次数相较于 MFF-1 并没有太多的改变，它的优化主要在于减少了平方根计算的次数。但是简单的利用个位判断减少的次数是有限的，还有更高的提升空间。

MFFV3 在 MFFV2 的基础上，可以利用aaa 和nnn 的最低位来计算b2b^2b2 的低位，相应减少了计算量。表上面xxx 就是aaa。

相较于 MFFV2，MFFV3 能够减少更多的平方根计算以加快速度。

MFFV4 在之前算法的基础上，通过 Y2MOD20 算法减少更多的计算次数。

# RSA 共模攻击 (RSA Commom Modulus Attack)

考虑这样子的一个情况：

{c1≡m1e(mod n)c2≡m2e(mod n)\begin{cases} c_1\equiv m^e_1(mod\ n)\\[2ex] c_2\equiv m^e_2(mod\ n) \end{cases}⎩⎨⎧​c1​≡m1e​(mod n)c2​≡m2e​(mod n)​

# 小公钥指数攻击 (RSA Low Public Exponent Attack)

# 中国剩余定理（CRT）

中国剩余定理是用来求解一次同余方程组的算法。

中国剩余定理：
正整数m1,m2,...,mkm_1,m_2,...,m_km1​,m2​,...,mk​ 两两互素，对于任kkk 个整数a1,a2,...,aka_1,a_2,...,a_ka1​,a2​,...,ak​，同余方程组：

{x≡a1(mod m1)x≡a2(mod m2)...x≡ak(mod mk)\begin{cases} x\equiv a_1(mod\ m_1)\\[2ex] x\equiv a_2(mod\ m_2)\\[2ex] ...\\[2ex] x\equiv a_k(mod\ m_k) \end{cases}⎩⎨⎧​x≡a1​(mod m1​)x≡a2​(mod m2​)...x≡ak​(mod mk​)​

必定有解且唯一，解为:

x≡∑i=1kMiMi−1ai(mod m)x\equiv \sum_{i=1}^kM_iM_i^{-1}a_i(mod\ m)x≡i=1∑k​Mi​Mi−1​ai​(mod m)

其中m=∏j=1kmj,∀1≤j≤k,Mj=mmj,Mj⋅Mj−1≡1(mod m)m=\prod^k_{j=1}m_j,\forall 1\le j\le k,M_j=\frac{m}{m_j},M_j\cdot M_j^{-1}\equiv 1(mod\ m)m=∏j=1k​mj​,∀1≤j≤k,Mj​=mj​m​,Mj​⋅Mj−1​≡1(mod m).

证明分为两个部分，第一部分证明解y0≡∑i=1kMiMi−1ai(mod m)y_0\equiv\sum_{i=1}^kM_iM_i^{-1}a_i(mod\ m)y0​≡∑i=1k​Mi​Mi−1​ai​(mod m) 的正确性。

因为m1,m2,...,mkm_1,m_2,...,m_km1​,m2​,...,mk​ 两两互素，所以必定有gcd(mj,Mj)=1gcd(m_j,M_j)=1gcd(mj​,Mj​)=1，也就是Mj−1M^{-1}_jMj−1​ 一定是存在的。Mj⋅Mj−1≡1(mod m),mi∣Mj(i≠j)M_j\cdot M_j^{-1}\equiv 1(mod\ m),m_i\mid M_j(i\neq j)Mj​⋅Mj−1​≡1(mod m),mi​∣Mj​(i=j)，那么

∀1≤j≤k,y0≡MjMj−1aj(mod mj)≡aj(mod mj)\forall 1\le j\le k,y_0\equiv M_jM_j^{-1}a_j(mod\ m_j)\equiv a_j(mod\ m_j)∀1≤j≤k,y0​≡Mj​Mj−1​aj​(mod mj​)≡aj​(mod mj​)

显然满足解的正确性。

接下来第二部分证明解的唯一性。假设存在两个解y0,y1y_0,y_1y0​,y1​，那么必定有∀1≤j≤k,y1≡y2(mod mj)\forall 1\le j\le k,y_1\equiv y_2(mod\ m_j)∀1≤j≤k,y1​≡y2​(mod mj​)，而m1,m2,...,mkm_1,m_2,...,m_km1​,m2​,...,mk​ 两两互素，有m=m1m2...mk=lcm(m1,m2,...,mk)m=m_1m_2...m_k=lcm(m_1,m_2,...,m_k)m=m1​m2​...mk​=lcm(m1​,m2​,...,mk​)，那么明显有y1≡y2(mod m)y_1\equiv y_2(mod\ m)y1​≡y2​(mod m)，也就证明了解的唯一性。

# 攻击实现

如果一个攻击者连续使用不同的模数n1,n2,...,nkn_1,n_2,...,n_kn1​,n2​,...,nk​ 和公钥指数eee 加密一系列相同的明文mmm 得到密文c1,c2,...,ckc_1,c_2,...,c_kc1​,c2​,...,ck​，同时公钥指数eee 并不大（e≤ke\le ke≤k）。

这样就会满足使用小公钥指数攻击的条件，我们注意到加密的结果满足：

{me≡c1(mod n1)me≡c2(mod n2)...me≡ck(mod nk)\begin{cases} m^e\equiv c_1(mod\ n_1)\\[2ex] m^e\equiv c_2(mod\ n_2)\\[2ex] ...\\[2ex] m^e\equiv c_k(mod\ n_k) \end{cases}⎩⎨⎧​me≡c1​(mod n1​)me≡c2​(mod n2​)...me≡ck​(mod nk​)​
