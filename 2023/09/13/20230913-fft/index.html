# 傅里叶变换 —— 连续到离散

高等数学的知识告诉我们，周期信号可以被分解成一系列正弦信号之和（当然要满足一定的条件，不过工科一般也碰不到吧），也就是所谓的傅里叶级数：

不管怎么样，傅里叶变换看起来都还是不错的。不过对于计算机来说却不是 —— 傅里叶变换处理连续的函数，得到连续的频谱。这显然难以通过计算机实现，因此，我们需要将其离散化。

首先，我们将时间轴离散化，将时间轴上的点看作是一个个的采样点，这样，我们就得到了一个离散的序列fN(k)f_N(k)fN​(k)。采用和傅里叶变换类似的定义，有：

如果考虑一个周期序列，周期序列对应的频谱是一个离散谱，那么如果能够将需要计算的序列fN(k)f_N(k)fN​(k) 转换为一个周期序列，就能够得到便于计算机处理的离散谱。

这便是离散傅里叶变换（DFT）和离散傅里叶逆变换（IDFT）。

# 快速傅里叶变换 —— 从多项式乘法开始

在引入快速傅里叶变换的概念之前，先来考虑一个问题 —— 如何快速地计算多项式乘法？

举个例子，现在有两个多项式：

A(x)=x3+2x2+3x+4B(x)=4x3+3x2+2x+1A(x) = x^3 + 2x^2 + 3x + 4\\ B(x) = 4x^3 + 3x^2 + 2x + 1A(x)=x3+2x2+3x+4B(x)=4x3+3x2+2x+1

我们要如何去计算这两个多项式的乘积C(x)=A(x)⋅B(x)C(x)=A(x)\cdot B(x)C(x)=A(x)⋅B(x)？

显然，直接计算不是不行 —— 我们不妨用更一般的形式来写：

A(x)=∑i=0naixiB(x)=∑i=0nbixiA(x)=\sum_{i=0}^{n}a_ix^i\\ B(x)=\sum_{i=0}^{n}b_ix^iA(x)=i=0∑n​ai​xiB(x)=i=0∑n​bi​xi

那么C(x)C(x)C(x) 可以表示为

C(x)=∑i=02ncixi,ci=∑j=0iajbi−jC(x)=\sum_{i=0}^{2n}c_ix^i,\\ c_i=\sum_{j=0}^{i}a_jb_{i-j}C(x)=i=0∑2n​ci​xi,ci​=j=0∑i​aj​bi−j​

很不幸，这也是个O(n2)\mathcal{O}(n^2)O(n2) 的算法。我们也需要寻找更好的算法。

但是，回头看一下上面的式子 —— 有什么特殊的地方？乘积多项式C(x)C(x)C(x) 的系数cic_ici​

ci=∑j=0iajbi−jc_i=\sum_{j=0}^{i}a_jb_{i-j}ci​=j=0∑i​aj​bi−j​

正好是多项式A(x)A(x)A(x) 和B(x)B(x)B(x) 系数的卷积！

卷积这件事会带来很多有趣的性质，卷积定理告诉我们 —— 时域相乘等于频域卷积，时域卷积等于频域相乘，这也让我们回到了傅里叶变换的问题上来。

不过暂且先抛开傅里叶变换，我们先把目光放到多项式上来。

# 系数表示和点值表示

我们前面采用了多项式的系数来表示它，不过这不算是唯一方法 —— 我们可以通过多项式在一些点上的取值来表示它。

为了后面介绍的方便，我们引入两个术语 ——Evaluation（估计）和 Interpolation（插值）。Evaluation 是指通过多项式的系数来计算多项式在某些点上的取值，而 Interpolation 则是指通过多项式在某些点上的取值来计算多项式的系数。

中学的知识就已经说明了，一个nnn 次多项式可以通过n+1n+1n+1 个点来唯一确定。因此，我们可以通过多项式在n+1n+1n+1 个点上的取值来表示它：

(xi,yi), i=0,1,⋯ ,n→A(x)(x_i,y_i),\ i=0,1,\cdots,n\rightarrow A(x)(xi​,yi​), i=0,1,⋯,n→A(x)

那么利用这种表示，我们也可以计算多项式的乘法：对于A(x)A(x)A(x) 上的一点(xi,ya,i)(x_i,y_{a,i})(xi​,ya,i​) 和B(x)B(x)B(x) 上的一点(xi,yb,i)(x_i,y_{b,i})(xi​,yb,i​)，我们可以计算出C(x)C(x)C(x) 上的一点(xi,ya,iyb,i)(x_i,y_{a,i}y_{b,i})(xi​,ya,i​yb,i​)。那么计算2N2N2N 个点，我们便可以通过拉格朗日插值法确定C(x)C(x)C(x)。

不过，这样的计算量依旧是O(n2)\mathcal{O}(n^2)O(n2) 的。

可以优化吗？

# 复数？

似乎不能。

还是举个例子。函数f(x)=x3+x2−x+1f(x)=x^3+x^2-x+1f(x)=x3+x2−x+1，我们需要选择四个点才能够唯一确定它。为了尽可能利用上面的性质，我们选择x1,−x1,x2,−x2x_1,-x_1,x_2,-x_2x1​,−x1​,x2​,−x2​ 这四个xxx 对应的点。

那我们便需要计算Peven(x2)=x2+1P_{even}(x^2)=x^2+1Peven​(x2)=x2+1 和Podd(x2)=x2−1P_{odd}(x^2)=x^2-1Podd​(x2)=x2−1 两个函数在x1,x2x_1,x_2x1​,x2​ 处的取值。拿Peven(x2)P_{even}(x^2)Peven​(x2) 为例，为了尽可能利用递归的性质，x2x_2x2​ 应该等于−x1-x_1−x1​。

但是，Peven(x2)P_{even}(x^2)Peven​(x2) 传入的参数是x2x^2x2，而x2=−x1x_2=-x_1x2​=−x1​，这样就会导致Peven(x22)=Peven(x12)P_{even}(x_2^2)=P_{even}(x_1^2)Peven​(x22​)=Peven​(x12​)，这样就无法利用递归的性质了。

如果我们需要计算的多项式次数为ddd，我们选择n=2⌈log⁡2d⌉n=2^{\lceil \log_2d\rceil}n=2⌈log2​d⌉（便于二分递归），我们要选的点便是xn=1x^n=1xn=1 的nnn 个根：

ωn/2+i=e−2jπ(n/2+i)/n=e−jπe−2jπii/n=−ωi\omega_{n/2+i}=e^{-2j\pi (n/2+i)/n}=e^{-j\pi}e^{-2j\pi i i/n}=-\omega_iωn/2+i​=e−2jπ(n/2+i)/n=e−jπe−2jπii/n=−ωi​

既然 Evaluation 和 DFT 是一致的，而 Interpolation 和 Evaluation 是相反的过程，那 Interpolation 和 IDFT 是否是一致的？
